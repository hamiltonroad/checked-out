## Harness Improvement Recommendations

| Brittle lookback in `check-eslint-overrides.sh` | Replace context-window grep with a real JSON parser (`jq`) that reads each override block's `files` field and only allows `max-lines: off` when the glob matches a documented-allow set. This is the same class of "stringly-typed config introspection" that should be rejected by reviewers going forward — add a CLAUDE.md note: "Harness scripts that introspect JSON/JS config MUST use a parser, not line-window grep." | No |

Bake this into `harness-health.sh` itself as a self-check. | No |
| Documented-only HARNESS-ENV-SINGLE-SOURCE | Add a registry-level convention: "Documentation-only rules MUST link to a tracking issue and MUST be re-evaluated within N days." `harness-health.sh` could parse a `Deferred-Until:` field and warn after the deadline. | No |
