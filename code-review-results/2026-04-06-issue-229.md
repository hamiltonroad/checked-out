| Medium | `frontend/eslint.config.js:~115` | The ESLint ban on direct mutation API calls only catches `fetch('/api/copies'|...)`. Stated intent in `standards/e2e-testing.md` §4 is broader than what the selector enforces. Either extend selectors to cover `page.request.*` / `request.fetch` / named session helpers, or scope the bullet in the standard to match the rule. | Architecture/Constraints | No |
| Medium | `frontend/eslint.config.js:~108` | The hardcoded-id rule `Property[key.name=/_id$/i][value.type='Literal']` only catches shorthand-key object properties (`{ copy_id: 1 }`). It bypasses string-keyed literals (`{ 'copy_id': 1 }`), call-site arguments (`checkoutCopy(1)`), array positional ids, and template-literal url params (`` `/books/${1}` ``). A spec author can still hardcode ids through any of those paths. Add a companion selector for `Property[key.type='Literal'][key.value=/_id$/i]` at minimum, and document the remaining escape hatches. | Architecture/Constraints | No |

Nothing enforces this. A spec that forgets the call leaks a checkout into the next run. The sibling helper `seedPatronWithHolds` has the same shape. Either (a) expose a disposer object (`{ …ids, cleanup }`) that can be wired into `afterEach`, or (b) register the cleanup on a Playwright fixture so teardown is automatic. | Verification/Back-pressure | No |
