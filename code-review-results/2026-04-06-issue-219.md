| High | `frontend/e2e/smoke/books-page-loads.spec.ts:9` | `EXPECTED_ERROR_PATTERNS` includes `/auth/i`, which silently swallows ANY console error containing the substring "auth" (e.g. legitimate `AuthProvider` failures, `useAuth` crashes, `authorization` errors). This defeats the purpose of the assertion `expect(consoleErrors).toEqual([])`. Narrow the allowlist to specific known-benign messages, or drop it entirely and assert on zero errors after a guaranteed-anonymous load. | Verification | No |
| High | `frontend/e2e/page-objects/BooksPage.ts:18` | `goto()` uses `page.waitForLoadState('networkidle')`. Playwright explicitly discourages `networkidle` ("DISCOURAGED ... Don't use this method for testing, rely on web assertions"). It is also flaky against React Query refetch intervals. Replace with an event-based wait such as `await this.getBookCards().first().waitFor({ state: 'visible' })` or a `waitForResponse` on the books API call. | Documentation | No |
| Medium | `frontend/e2e/smoke/login-flow.spec.ts:15` | Token cookie lookup `cookies.find((c) => /token/i.test(c.name))` is too loose — matches any cookie with "token" in its name (CSRF token, etc.). Use the exact cookie name (e.g. `accessToken` or whatever `auth.js` sets) imported from a constant so the test fails loudly if the cookie is renamed. | Architecture | No |
| Medium | `frontend/e2e/smoke/login-flow.spec.ts:23` | URL assertion `expect(page.url()).toMatch(/\/$/)` matches every URL ending in `/` (including `http://localhost:5173/login/`). Acceptance criterion requires "landing URL is the Books page" — assert against the exact path with `expect(page).toHaveURL(/\/$/)` after `new URL(...).pathname === '/'`, or compare `pathname` directly. | Verification | No |
| Medium | `frontend/e2e/smoke/login-flow.spec.ts:19` | After `loginAs(page, 'patron')` the test calls `booksPage.goto()`, which re-navigates to `/`. This bypasses the acceptance criterion "landing URL is the Books page" — the navigation under test is the post-login redirect, not a manual `goto`. Assert the redirect happened *before* navigating again, e.g. `await page.waitForURL('/')` immediately after `loginAs`. | Verification | No |
| Medium | `frontend/e2e/smoke/book-detail-modal.spec.ts:21` | Acceptance criterion calls for "per-format availability **and status badge**". The test asserts `Copies`, `Genre`, and `/of \d+ available/`, but never asserts the status badge. Add an assertion for the status badge element (e.g. `available`/`checked out` chip). | Verification | No |
| Low | `frontend/e2e/page-objects/BooksPage.ts:32` | `openBookDetail` does `card.waitFor({ state: 'visible' })` then `card.click()` — Playwright's auto-waiting on `click()` already handles visibility. The explicit `waitFor` is redundant. | Documentation | No |
| Low | `frontend/e2e/smoke/books-page-loads.spec.ts:35` | `await expect(cards.first()).toBeVisible()` followed by `expect(await cards.count()).toBeGreaterThan(0)` is redundant — the first assertion already implies count > 0. Drop one. | Documentation | No |
